### F08: llama.cpp b9993 — Hy3 Architecture + MTP Speculative Decoding
**Date:** July 13, 2026 (released)
**Signal:** `#SIGNAL`
**Evidence Tier:** T2 | **Confidence:** 75%
**Sources:**
- [GitHub: ggml-org/llama.cpp b9993](https://github.com/ggml-org/llama.cpp/releases/tag/b9993)
- [PR #25395: Hy3 support with MTP speculative decoding](https://github.com/ggml-org/llama.cpp/pull/25395)

**llama.cpp** released **b9993**, adding full support for **Tencent's Hy3 (hy_v3)** architecture — a MoE decoder stack with per-head Q/K RMSNorm, sigmoid router with expert selection bias, and an always-active ungated shared expert. Key details:
- Ported from charlie12345's ROCmFPX fork (src/models/hyv3.cpp)
- **MTP speculative decoding** enabled for Hy3 models
- macOS arm64, iOS XCFramework, Linux x64/arm64/s390x, Windows CUDA 12/13/Vulkan/OpenVINO/SYCL/HIP builds
- The base implementation follows current mainline APIs (hparams.n_layer(), build_qkv, build_moe_ffn with fused gate_up + scale tensors)

**Why it matters:** Hy3 support means Karl can now run Tencent's Hunyuan 3 architecture locally via llama.cpp — expanding the local model catalog. MTP speculative decoding is a meaningful inference speedup for MoE models. For an M4 Max: check whether GGUF quants of Hy3 models exist and fit in unified memory.

### F09: Mesh LLM — Distributed Inference, Pooled GPUs → Single API Endpoint
**Date:** July 11, 2026 (launched)
**Signal:** `#SIGNAL`
**Evidence Tier:** T1 | **Confidence:** 60%
**Sources:**
- [DailyAIWorld: Mesh LLM Distributed Inference](https://dailyaiworld.com/blogs/mesh-llm-distributed-inference-2026)
- [HN Discussion (item 48876505)](https://news.ycombinator.com/item?id=48876505)

**Mesh LLM** is an open-source distributed inference system that pools GPU resources across multiple machines into a single OpenAI-compatible API endpoint at `localhost:9337/v1`. Uses **iroh's QUIC-based P2P networking** with automatic NAT traversal. Key details:
- Exposes one unified endpoint regardless of how many nodes participate
- Handles model distribution across peers automatically
- Early-stage but the HN discussion reports first-try success with `mesh-llm --auto`
- Models up to 235B parameters (per source)

**Why it matters:** This is a novel approach to local inference at scale — instead of running one big model on one machine, pool spare GPUs across your network. For Karl's M4 Max: potentially useful for running models that exceed single-machine VRAM by pooling with other nodes. The iroh P2P layer means no central server required. Worth watching as a pattern for local-first distributed inference.

Muse Spark 1.1 was built specifically for agentic coding. Gemini 3.5 Pro targets autonomous workflows. The "coding agent" is becoming the default AI product category — and local models (Qwen3-Coder, Hy3) are closing the gap on API models for routine tasks.
