## Tier S — Breakthroughs (paradigm-level signals)

### S1: DiffusionGemma — First Diffusion LLM Goes Open
**Signal:** `#BREAKTHROUGH` | **Date:** June 10, 2026 | **Sources:** blog.google/innovation-and-ai/technology/developers-tools/diffusion-gemma-faster-text-generation, huggingface.co/google/diffusiongemma-26B-A4B-it, vllm.ai/blog/2026-06-10-diffusion-gemma, blogs.nvidia.com/blog/rtx-ai-garage-local-gemma-diffusion, arstechnica.com
**Confidence:** 90% ⭐

Google DeepMind released **DiffusionGemma**, a 26B-parameter Mixture-of-Experts model (4B active per forward pass) that generates text via diffusion — iteratively denoising a 256-token canvas in parallel — instead of standard autoregressive decoding. Apache 2.0 license.

**Key details:**
- **Architecture:** Built on the Gemma 4 backbone with a diffusion head. Encoder mode uses causal attention for prefill; decoder mode uses bidirectional attention for denoising. Self-conditioning feeds back softmax distributions between steps. Entropy-bound rule decides which positions to keep at each step.
- **Performance:** Up to 1,000 tok/s on NVIDIA H100, 700+ tok/s on RTX 5090, 150 tok/s on DGX Spark — roughly 4x faster than equivalent autoregressive models in single-user workloads. Fits in ~18GB VRAM at Q4_K_M quantization.
- **Multimodal:** Processes interleaved text, image, and video inputs (256K context window, 140+ languages). Supports Gemma 4-style thinking mode with <|think|> token.
- **vLLM integration:** First diffusion LLM natively supported in vLLM via model runner v2's ModelState abstraction. Unsloth Studio support WIP. llama.cpp PR #24423 (entropy_bounded_denoising sampler) needed for standard Ollama use.
- **Quality tradeoff:** Google explicitly says "for maximum quality production work, deploy standard Gemma 4." Benchmark deltas: MMLU Pro 77.6% vs 82.6%, GPQA Diamond 73.2% vs 82.3%, AIME 2026 69.1% vs 88.3%. But edges on HLE no tools (11.0% vs 8.7%).
- **How it works:** Instead of predicting one token at a time, DiffusionGemma generates text the way diffusion image models work — starts from random noise and iteratively refines a full block. Bidirectional attention lets every token attend to every other token in the canvas simultaneously.

**Karl-relevance:** HIGH. This is genuinely novel architecture — the first usable diffusion LLM on consumer hardware (18GB VRAM sweet spot). For agentic loops, interactive chat, and code infilling where latency kills UX, this could be transformative. The vLLM integration means it plugs into the entire open-source serving ecosystem. Not yet optimized for Apple Silicon, but the architecture is worth watching closely.

**Evidence:** T3 (Google official blog post, HF model card with Apache 2.0 tag). T2 (NVIDIA blog confirming RTX/DGX optimization, vLLM integration blog, Unsloth docs). Independent verification: HF confirms model exists at google/diffusiongemma-26B-A4B-it, created 2026-06-09T12:40Z, tags include `diffusion_gemma`, `image-text-to-text`, `conversational`, `license:apache-2.0`.
