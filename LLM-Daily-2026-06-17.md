**Benchmark Table (selected):**

| Benchmark | GLM-5.2 | GPT-5.5 | Claude Opus 4.8 |

**Architecture:** MoE with 753B total parameters. Key innovation: **IndexShare** — one indexer reused across every four sparse attention layers, reducing per-token FLOPs by 2.9× at 1M context. Upgraded Multi-Token Prediction (MTP) layer adds up to +20% accepted token length during speculative decoding. Two thinking modes: Max (deeper reasoning, ~85K output tokens/task) and High (half the token cost for near-identical scores).

**Significance:** This is the first time a Chinese open-weight model has independently beaten GPT-5.5 on long-horizon coding benchmarks. The MIT license means Karl can download and run it locally today. Combined with the 1M context window, this makes GLM-5.2 a serious local-first candidate for agentic coding tasks — particularly relevant to an M4 Max with 128GB unified memory.
