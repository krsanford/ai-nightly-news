### F01: Meta Muse Spark 1.1 — Multimodal Agentic Coding Model Goes Public
**Date:** July 9–10, 2026
**Signal:** `#BREAKTHROUGH`
**Evidence Tier:** T2 | **Confidence:** 85%
**Sources:**
- [TechCrunch](https://techcrunch.com/2026/07/09/meta-enters-the-crowded-ai-coding-battle-with-muse-spark-1-1/)
- [Meta AI Blog](https://ai.meta.com/blog/introducing-muse-spark-meta-model-api/)
- [Reuters via ETEnterpriseai](https://enterpriseai.economictimes.indiatimes.com/news/industry/meta-unveils-muse-spark-11-revolutionizing-ai-development-for-coders/132306494)

Meta publicly released **Muse Spark 1.1**, described as its "most capable model for agentic and coding work" (Alexandr Wang, head of AI at Meta). Key specs:
- **Multimodal** (text, images, video) with a **1M token context window**
- **Agentic focus**: multistep reasoning, tool use, computer use, code generation/debugging/migration
- **Pricing**: $1.25 per million input tokens / $4.25 per million output tokens — competitive with Haiku 4.5 and GPT-5.6 Luna, below Sonnet 4.6
- **API**: Meta Model API now in public preview ($20 free credits for developers)
- **Availability**: U.S. public preview; partners had access since April private preview

Zuckerberg called it "a strong agentic and coding model at a very low price" on X — his first post there in three years. He also teased "more to come soon." Meta is also building a next-gen model codenamed **Watermelon**.

**Why it matters:** This is Meta's first serious attempt to compete directly with OpenAI (Codex) and Anthropic (Fable 5) in the agentic coding space. The pricing strategy signals an intent to win developer mindshare through cost, then monetize at scale. For Karl: if Meta follows its Llama pattern (open-source after proprietary), this could become a locally-runnable model. The multimodal + agentic combination is the frontier where all three labs are converging.

**Updated:** Muse Image (Meta's first image-generation model) also launched July 7, making this a two-pronged creative+coding push.

### F04: Ollama $65M Series B — 8.9M Developers, GPU-Time Cloud
**Date:** July 9–10, 2026
**Signal:** `#SIGNAL`
**Evidence Tier:** T2 | **Confidence:** 85%
**Sources:**
- [TechCrunch](https://techcrunch.com/2026/07/09/popular-open-source-ai-developer-tool-ollama-raises-65m-grows-to-nearly-9m-users/)
- [TechTimes Deep Dive](https://www.techtimes.com/articles/320061/20260710/ollama-closes-65m-series-b-reaches-89m-developers-local-open-weight-ai.htm)

Ollama closed a **$65M Series B** led by Theory Ventures (Benchmark, 8VC, Y Combinator, Pace Capital, 49 Palms, GTMFund participated). Total funding: **$88M**. Key stats:
- **8.9 million monthly active developers** (1 million new installs/week)
- **14-person team** that built the largest open-model developer network
- **85% of Fortune 500** companies use Ollama (healthcare, finance, government)
- **Core inference engine**: llama.cpp (C/C++ with GGUF format), OpenAI-compatible REST API
- **Cloud shift**: GPU-time billing (not per-token) for models exceeding local VRAM
- **Hybrid architecture**: local runs on hardware; cloud routes to same API, no config change

**Why it matters:** This is infrastructure-level validation of the local-first approach. 8.9M developers with 14 people is a stunning efficiency ratio. The GPU-time billing model is a direct response to the agentic workload economics (hundreds of sequential tool calls per session). For Karl: Ollama's growth validates his stack choice, and the cloud fallback means heavy models (Nemotron, GLM, DeepSeek) are accessible without local hardware.

### F05: llama.cpp v0.17.1 — ET Backend + Security Fixes
**Date:** July 10, 2026
**Signal:** `#SIGNAL`
**Evidence Tier:** T1 | **Confidence:** 70%
**Sources:**
- [GitHub Releases](https://github.com/ggml-org/llama.cpp/releases)

llama.cpp v0.17.1 released with significant work:
- **ggml-et (TensorFMA backend)**: Initial ET backend ([#24179](https://github.com/ggml-org/llama.cpp/pull/24179)), including MUL_MAT, ROPE, RMS_NORM, GLU, SOFT_MAX kernels — a new compute path for Apple Silicon and similar hardware.
- **FlashAttention optimizations**: Vectorized RoPE, parallel softmax by rows, FA_F16 using TensorFMA.
- **Security**: CVE-2026-7482 "Bleeding Llama" (CVSS 9.1) patched — GGUF model loader vulnerability from May 2026.

**Why it matters:** The ET backend represents a new inference path that could significantly impact performance on Apple Silicon hardware (Karl's M4 Max). The security fix is critical for anyone running GGUF models.
