### Ornith-1.0: Self-Scaffolding LLMs for Agentic Coding (DeepReinforce)
**Signal:** #BREAKTHROUGH — A genuinely novel training paradigm, not just another benchmark-chaser. First model release from DeepReinforce (a new lab). Built on Gemma 4 + Qwen 3.5 foundations (both Apache 2.0), released MIT-licensed. The key innovation: a self-improving training framework where the model jointly learns to solve tasks AND construct the task-specific scaffolds/harnesses that guide those solutions — instead of relying on human-designed harnesses.

**What it does:**
- Four sizes: **9B Dense** (edge), **31B Dense** (upcoming), **35B MoE**, and **397B MoE** (flagship).
- Ornith-1.0-397B: 77.5 on Terminal-Bench 2.1 (Terminus-2), 82.4 on SWE-Bench Verified — beating Claude Opus 4.7 (70.3 / 80.8). Also beats MiniMax M3 and DeepSeek-V4-Pro on both benchmarks.
- Ornith-1.0-35B: 64.2 on TB-2.1, surpassing Qwen3.5-397B (53.5) despite being ~10x smaller in params.
- Ornith-1.0-9B: 43.1 on TB-2.1 (Terminus-2), matching or exceeding Gemma 4-31B — deployable on edge devices and single-GPU setups.
- Simon Willison reports running the 35B GGUF (~20GB) via LM Studio with Pi, handling multi-turn agent harnesses over tool calls proficiently.
- **Karl-relevance:** HIGH. The 35B variant fits comfortably on a single GPU (RTX 4090-class). Self-scaffolding means the model gets better at coding tasks through its own training loop — this is an open-weight, locally-runnable model that could meaningfully compete with Opus 4.8 for agentic coding workloads on Karl's M4 Max. MIT license allows commercial use and fine-tuning.

### vLLM Semantic Router: Micro-Agent Runtime
**Signal:** #SIGNAL — vLLM announced the Semantic Router as a "bounded micro-agent runtime" for AI inference. The router can turn one model API call into a bounded collaboration inside the serving layer, implementing patterns like:
- **Confidence**: sequential escalation (cheaper first, escalate only when score is low)
- **Ratings**: parallel fan-out with hard concurrency cap + rating-aware aggregation
- **ReMoM**: repeated mixture-of-model reasoning with quorum-based synthesis
- **Fusion**: panel-judge-final pattern where disagreement becomes evidence
- **Workflows**: bounded micro-agent role system (planner/verifier/finalizer)

**Karl-relevance:** HIGH. This directly maps to how Karl's OpenClaw setup routes between models. The Semantic Router is an open serving primitive that could replace hand-written agent orchestration with a configurable router. For local inference on M4 Max, the "confidence" pattern (try cheaper first, escalate only when needed) is particularly relevant for cost-aware routing.
