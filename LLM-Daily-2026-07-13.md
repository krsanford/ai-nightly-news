### F06: ChatGPT Work — OpenAI's Agentic Desktop App
**Date:** July 9, 2026 (announced)
**Signal:** `#SIGNAL`
**Evidence Tier:** T2 | **Confidence:** 75%
**Sources:**
- [OpenAI: ChatGPT for Your Most Ambitious Work](https://openai.com/index/chatgpt-for-your-most-ambitious-work/)
- [ChatGPT Desktop App Docs](https://learn.chatgpt.com/docs/app)
- [Skills in ChatGPT (Help Center)](https://help.openai.com/en/articles/20001066-skills-in-chatgpt)

OpenAI launched **ChatGPT Work** on July 9 — a new desktop app for macOS and Windows that combines ChatGPT with Codex into a single workspace. Key capabilities:
- **Multi-agent parallel execution:** Run multiple coding agents in parallel, organized by project and thread
- **Long-horizon tasks:** Stay with a project for hours, pulling context from connected apps and files
- **Deliverable production:** Turn goals into finished sheets, slides, docs, and web apps (not just discuss them)
- **Skills system:** Follows the Agent Skills open standard; Personal Skills for Business/Enterprise/Edu users
- **5M+ weekly active users** on Codex alone
- **Atlas Browser retired** (July 9) to fold features into the ChatGPT desktop app

**Why it matters:** This is OpenAI's answer to Anthropic's Claude Cowork — and it's a platform play, not just a model release. The "finished deliverables" angle shifts the surface from chatbot to agent workspace. For Karl: the multi-agent parallel pattern and the Agent Skills open standard are directly relevant to OpenClaw's skill framework.

### F07: Muse Spark 1.1 — Meta's First Paid Model Pivot
**Date:** July 9, 2026 (released)
**Signal:** `#SIGNAL`
**Evidence Tier:** T2 | **Confidence:** 75%
**Sources:**
- [Build with Muse Spark — Meta Developer Blog](https://developer.meta.com/ai/resources/blog/build-with-muse-spark/)
- [Muse Spark 1.1 — Product Hunt](https://www.producthunt.com/products/muse-spark-1-1-by-meta)

Meta released **Muse Spark 1.1** on July 9 — a multimodal reasoning model built specifically for agentic tasks. Key details:
- **Agentic focus:** Coding, multi-step reasoning, tool use, computer use, video understanding
- **Pricing:** $1.25/million input tokens via the new Meta Model API (aggressive pricing)
- **API:** OpenAI-compatible; model ID `muse-spark-1.1`
- **Benchmark claims:** Meta claims it beats similar products from OpenAI, Anthropic, and Google
- **Strategic significance:** First paid model from Meta — a break from the open-weight Llama era

**Why it matters:** This is the first time Meta has moved from "give away open weights" to "charge for API access." The $1.25/million token price is aggressively below industry standard (GPT-5.6 Sol: $5/$30, Claude Fable 5: $10/$50). For Karl: worth testing as a cost-effective agentic backend. Meta's strategic pivot signals the agent war is moving from "who has the smartest model" to "who owns the cheapest inference path."

## SYNTHESIS & CROSS-FINDING CONNECTIONS

**The July 9 Triopoly Launch Was Real:** OpenAI (ChatGPT Work + GPT-5.6), Meta (Muse Spark 1.1), and Anthropic (Fable 5 extension) all moved on or around July 9. This is not a coincidence — the coding-agent war has moved from model benchmarking to platform ownership. OpenAI built the surface (desktop app with parallel agents); Meta built the cheapest inference path ($1.25/million tokens); Anthropic bought time (Fable 5 extensions) while developing Honeycomb EAP. The three approaches — platform, price, and stealth R&D — represent the full competitive spectrum.

**The Open-Weight vs. API Divide Is Hardening:** Meta's Muse Spark 1.1 is the first paid model from a company built on open-weight philosophy. This signals that even the "open" labs are converging on paid API access as the primary revenue model. For Karl: the local-first advantage (Qwen3-Coder + NVFP4) is getting more valuable precisely because the frontier labs are moving to paid APIs.

**ChatGPT Work vs. Claude Cowork — The Surface War:** Both products target the same use case (agentic desktop automation), but from different angles. OpenAI's approach: combine chat + code agent into one workspace with parallel execution. Anthropic's: scheduled tasks that run while you're gone, 90% non-coding usage. The data suggests Anthropic's users are doing more operational work (business ops, content creation) while OpenAI's is coding-heavy. For Karl: this distinction matters for what surface to build on.
