### F01 — Gemma 4 Family Released (June 3, 2026)
**Source:** Google DeepMind — https://blog.google/innovation-and-ai/technology/developers-tools/introducing-gemma-4-12B/

Google released a full Gemma 4 family — five model sizes spanning edge to server:
| Model | Params | Active | Architecture | Modalities | Context |
|-------|--------|--------|-------------|------------|---------|
| E2B | 2.3B eff (5.1B w/ embeddings) | - | Dense + PLE | Text, Image, Audio | 128K |

| **31B Dense** | 30.7B | - | Dense | Text, Image | 256K |

Key innovations: encoder-free architecture on the 12B (raw image patches + audio waveforms projected directly into LLM embedding space via lightweight linear layers), hybrid attention (local sliding window + full global with Proportional RoPE), native system prompt support, configurable thinking modes. Apache 2.0 license. GGUF quantizations already available via Unsloth. Ollama models: `gemma4:e2b`, `gemma4:e4b`, `gemma4:12b`, `gemma4:12b-mlx`, `gemma4:26b`, `gemma4:31b`, `gemma4:31b-cloud`.

### F02 — DiffusionGemma 26B-A4B Released (June 10, 2026)
**Source:** Google DeepMind — https://deepmind.google/models/gemma/diffusiongemma/
**Community site:** https://diffusiongemma.dev/

An experimental open-weights model using discrete diffusion instead of token-by-token autoregressive decoding. Same Gemma 4 26B A4B MoE backbone (25.2B total, 3.8B active), but generates 256-token canvases in parallel with iterative denoising. Up to 4x faster throughput for single-user/low-batch workloads. vLLM native support on day 0; llama.cpp PR #24427 under review (not merged). Lower quality than autoregressive Gemma 4 baseline across reasoning, coding, and multimodal evals — it's a speed tradeoff, not a clean quality upgrade.

**Evidence:** DeepMind model page, TechTimes coverage, Nerova.ai deep-dive, DiffusionGemma.dev community site with benchmarks.
**Triage:** T0 (breakthrough). First open diffusion-language-model to give the field a real test case. Interesting architecture but clearly positioned as experimental. The speed advantage collapses on low-compute GPUs (3060/4060/laptops) and Apple Silicon per early reports.
